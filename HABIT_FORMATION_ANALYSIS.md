## HOOK MODEL ANALYSIS

### Trigger → Action → Reward → Investment

#### Morning Security Check Habit

**External Trigger:**

- Desktop notification: "☀️ Good morning! Your security score: 85/100"
- Time: 9:00 AM local time

**Internal Trigger:**

- Anxiety: "Did I deploy something risky yesterday?"
- Curiosity: "What's my security score today?"

**Action:**

- Click notification → Opens VS Code sidebar
- One-click: "Run morning scan"
- Time to complete: < 10 seconds

**Variable Reward:**

- Sometimes: Clean bill of health (relief)
- Sometimes: Found 2 issues (discovery + hero moment)
- Sometimes: New record streak (achievement)

**Investment:**

- Data: Scan history, custom collections
- Social: Team security score
- Customization: Alert thresholds

#### Pre-Deploy Check Habit

**External Trigger:**

- Git pre-commit hook
- VS Code command palette: "RakshEx: Pre-deploy scan"

**Internal Trigger:**

- Fear: "Am I about to push a secret?"
- Professionalism: "I check before every deploy"

**Action:**

- Keyboard shortcut (configurable)
- Scan runs automatically
- Results in < 5 seconds

**Variable Reward:**

- Clean: Green checkmark, "Good to go! 🚀"
- Issues: Specific fix instructions

**Investment:**

- Time: 2 seconds per check × 10 deploys/day = 20s investment
- Reputation: "I never push secrets"

## ANTI-HABITS TO ELIMINATE

| Anti-Habit             | Why It's Bad                   | Fix                        |
| ---------------------- | ------------------------------ | -------------------------- |
| Too many notifications | Creates notification blindness | Max 1/day, only high-value |
| Slow scans             | Breaks flow                    | < 5s target                |
| False alarms           | Destroys trust                 | Improve accuracy to > 90%  |
| Complex UI             | Adds friction                  | One-click actions          |
| No clear CTA           | Users don't know what to do    | Contextual prompts         |
