## ANTICIPATED QUESTIONS

### The Traction Question

**Q:** "What traction do you have?"

**Strong answer:**
"[N] active developers, [X]% weekly growth rate,
[Y] scans run per week, [Z]% D7 retention.

Most importantly: [specific story of user who found
real value — e.g., 'One user found a $50K/year
security issue in their first scan.']"

**Weak answer:**
"We have a lot of interest." (vague, no proof)

---

### The Market Question

**Q:** "How big is this market?"

**Strong answer:**
"Today: 30M developers building with AI.
Annual AI API spend: $50B+ and growing 3× YoY.

In 3 years: Every developer will need AI Runtime Protection,
just like every developer needs API security today.

Bottom-up: $50/user/mo × 1M developers = $600M ARR.
Top-down: 1% of $50B API spend = $500M."

---

### The Moat Question

**Q:** "What's your moat?"

**Strong answer:**
"Four things:

1. Workflow lock-in — we're inside VS Code where developers live.
2. Data flywheel — every scan makes our detection better.
3. First-mover in a new category — we define 'AI Runtime Protection.'
4. Founder-market fit — I lived this problem for [X] months."

---

### The Why-Now Question

**Q:** "Why is this the right time?"

**Strong answer:**
"Three reasons:

1. AI agents just became real — millions deployed in production.
2. No existing security tool understands AI agent behavior.
3. Enterprise AI security budgets are exploding but have nowhere to go.

If we don't do this now, Snyk or Datadog will in 12 months."

---

### The Founder Question

**Q:** "Why are you the right team?"

**Strong answer:**
"I'm a developer who spent [X] months building AI agents
and got burned by surprise costs and security issues.

I built RakshEx to solve my own problem.
[Technical depth — e.g., built the first version solo in 3 months].

I've also [relevant experience — e.g., shipped products at
previous companies, open source contributions, etc.]."

---

### The Competition Question

**Q:** "What about Snyk/Datadog/Portkey?"

**Strong answer:**
"They solve adjacent problems:

- Snyk: Code security (we complement them)
- Datadog: Infrastructure monitoring (different layer)
- Portkey: Cost tracking (we do security + cost + auto-stop)

None of them auto-stop agents or detect hidden costs
in AI-specific ways. We're the only AI Runtime Protection tool."

| Objection                 | Response                                                                              |
| ------------------------- | ------------------------------------------------------------------------------------- |
| "Too early"               | "We have [N] users actively using us weekly. We're not pre-launch."                   |
| "Too niche"               | "Every developer using AI will need this. That's 30M people today, growing to 100M+." |
| "Big companies will copy" | "That's validation. But we're inside their IDE and have the data flywheel."           |
| "No revenue"              | "We're in beta. [N]% of users said they'd pay. We're about to test pricing."          |
| "Solo founder"            | "I built this to MVP solo. YC helps me find the right co-founder faster."             |
| "Not technical enough"    | "I've shipped [X] production systems. [Specific technical achievement]."              |

- [ ] Don't use notes
- [ ] Record yourself

- [ ] Write down questions you struggled with
- [ ] Refine your answers
- [ ] Practice those answers 5 more times
