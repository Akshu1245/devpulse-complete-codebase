# RakshEx VS Code Marketplace Review Strategy

> How to get 5-star reviews from beta users.
> Date: 2026-05-17

---

## REVIEW IMPORTANCE

### Why Reviews Matter

- VS Code Marketplace uses reviews in search ranking
- 4.5+ stars = significantly more installs
- Reviews with screenshots rank higher
- First 20 reviews set the tone permanently

### Target

| Milestone        | Target   | Timeline       |
| ---------------- | -------- | -------------- |
| First 10 reviews | 5 stars  | Week 1 of beta |
| 20 reviews       | 4.8+ avg | Week 2         |
| 50 reviews       | 4.7+ avg | Month 1        |
| 100 reviews      | 4.5+ avg | Month 2        |

---

## WHEN TO ASK FOR REVIEWS

### Trigger: "Aha Moment"

Ask for a review immediately after the user experiences value:

| Scenario                     | Timing        | Message                                                                                |
| ---------------------------- | ------------- | -------------------------------------------------------------------------------------- |
| Found first secret           | Within 5 min  | "We found an exposed key! Help other developers stay safe too — leave a quick review?" |
| AgentGuard stopped something | Within 5 min  | "AgentGuard just saved you money! Would you share your experience?"                    |
| Completed 3rd scan           | After scan    | "You're a power user! Your review helps us reach more developers."                     |
| Weekly summary viewed        | After viewing | "You've been protected for a week. Mind sharing your thoughts?"                        |

### Never Ask When:

- User just encountered an error
- User marked findings as false positives
- User hasn't run a scan yet
- User uninstalled the extension

## REVIEW QUALITY TACTICS

### Encourage Detailed Reviews

Bad: "Great tool!" (1 sentence)
Good: "RakshEx found $400 in hidden reasoning costs and 3 exposed secrets in our API collection. The VS Code integration is seamless. Highly recommend for any team using LLMs." (3+ sentences, mentions specific value)

### How to Get Detailed Reviews:

1. **Ask specific questions:** "What did RakshEx find that surprised you?"
2. **Share examples:** "Here's what other users wrote..."
3. **Offer to draft:** "Happy to draft something you can edit"
4. **Incentivize:** Best review of the week gets featured + swag

### Review Content Wishlist

| Element                  | Why It Helps                     |
| ------------------------ | -------------------------------- |
| Mention specific feature | Helps SEO for that feature       |
| Include dollar amount    | Social proof for cost savings    |
| Mention company size     | Helps other similar users decide |
| Compare to alternative   | Positions vs competitors         |
| Mention onboarding speed | Reduces friction for new users   |
