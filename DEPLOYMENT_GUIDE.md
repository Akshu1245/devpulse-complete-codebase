# RakshEx Production Deployment Guide

> Complete step-by-step guide to take RakshEx from local dev to live production.

---

## Phase 1: Backend on Render (30 minutes)

### Step 1: Add Payment Method to Render

1. Go to https://dashboard.render.com
2. Click your profile (top right) → **Billing**
3. Click **Add Payment Method**
4. Enter card details (any major credit/debit card works)
5. Verify with OTP if required

### Step 2: Deploy via Blueprint

```bash
cd /path/to/RakshEx_Complete_Codebase
render login                    # Already done
render blueprints apply --confirm
```

This creates:

- `rakshex-backend` web service (Docker, auto-deploy from `feature/push-current-work`)
- `rakshex-redis` Redis instance

### Step 3: Create MySQL Database

Render blueprints do **not** support managed MySQL. You have three options:

**Option A: Render MySQL (manual)**

1. Go to https://dashboard.render.com → **New** → **MySQL**
2. Name: `rakshex-db`
3. Plan: Starter ($7/mo) or Standard ($15/mo)
4. Region: Choose closest to your users (Singapore for India)
5. Wait for "Available" status
6. Copy the **External Connection String**

**Option B: PlanetScale (serverless MySQL)**

1. Go to https://planetscale.com → Sign up
2. Create database: `rakshex`
3. Create deploy request (main branch)
4. Copy connection string from **Connect** → **Node.js** → `DATABASE_URL`

**Option C: Railway / Neon**

- Railway: https://railway.app → New Project → Add MySQL
- Neon: https://neon.tech → Create project → Copy connection string

### Step 4: Set Environment Variables

In Render Dashboard → `rakshex-backend` → **Environment**:

```
DATABASE_URL=mysql://username:password@host:3306/rakshex
REDIS_URL=(auto-populated by blueprint)
JWT_SECRET=(auto-generated by blueprint)
APP_URL=https://rakshex-backend-xxx.onrender.com
NEXT_PUBLIC_APP_URL=https://app.rakshex.in
SMTP_HOST=smtp.sendgrid.net
SMTP_PORT=587
SMTP_USER=apikey
SMTP_PASS=SG.xxx.your-sendgrid-api-key
SMTP_FROM=noreply@rakshex.in
GITHUB_APP_ID=
GITHUB_APP_PRIVATE_KEY=
GITHUB_WEBHOOK_SECRET=
SENTRY_DSN=
RAZORPAY_KEY_ID=
RAZORPAY_KEY_SECRET=
```

### Step 5: Run Database Migrations

```bash
# SSH into Render service (via Dashboard → Shell)
# Or run locally with production DATABASE_URL
export DATABASE_URL="your-production-url"
npx drizzle-kit migrate
```

### Step 6: Verify Health Check

```bash
curl https://rakshex-backend-xxx.onrender.com/api/health
# Expected: {"status":"ok","timestamp":"..."}
```

## Phase 2: Frontend on Vercel (10 minutes)

Already deployed! Just update the API URL:

1. Go to https://vercel.com/dashboard → `rakshex-frontend`
2. **Settings** → **Environment Variables**
3. Update:
   ```
   NEXT_PUBLIC_TS_API_URL=https://rakshex-backend-xxx.onrender.com
   ```
4. **Redeploy** (Vercel → Deployments → Redeploy latest)

## Phase 3: Domain & DNS (20 minutes)

### Buy Domain

**Recommended:**

- Namecheap (https://namecheap.com) — $8-12/yr for .in
- GoDaddy India — Often has .in promotions
- Cloudflare Registrar — At-cost pricing

**Search:** `rakshex.in` or `rakshex.io` or `getrakshex.com`

### Configure DNS

**A Records:**

```
Type: A
Name: @
Value: 76.76.21.21    # Vercel's IP
TTL: 300
```

**CNAME for www:**

```
Type: CNAME
Name: www
Value: cname.vercel-dns.com
TTL: 300
```

**API subdomain:**

```
Type: CNAME
Name: api
Value: rakshex-backend-xxx.onrender.com
TTL: 300
```

### Add Domain to Vercel

1. Vercel Dashboard → `rakshex-frontend` → **Settings** → **Domains**
2. Add `rakshex.in` and `www.rakshex.in`
3. Follow Vercel's DNS verification steps

### Update Environment Variables

After domain is live, update all `APP_URL` references:

**Render:**

```
APP_URL=https://api.rakshex.in
NEXT_PUBLIC_APP_URL=https://rakshex.in
```

**Vercel:**

```
NEXT_PUBLIC_APP_URL=https://rakshex.in
NEXT_PUBLIC_TS_API_URL=https://api.rakshex.in
```

## Phase 4: Email Setup (15 minutes)

### SendGrid (Recommended)

1. Sign up at https://sendgrid.com
2. Complete sender authentication:
   - Add domain: `rakshex.in`
   - Add CNAME records to DNS (SendGrid provides them)
   - Wait for verification (can take up to 24 hours)
3. Create API key: **Settings** → **API Keys** → **Create API Key** (Full Access)
4. Copy API key to Render env var `SMTP_PASS`

### Mailgun (Alternative)

1. Sign up at https://mailgun.com
2. Add domain: `mg.rakshex.in`
3. Verify domain with DNS TXT records
4. Copy API key to `SMTP_PASS`

### Test Email Delivery

```bash
curl -X POST https://api.rakshex.in/api/trpc/auth.signup \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"Test123!"}'
# Check inbox for welcome email
```

## Phase 5: Monitoring & Alerting (20 minutes)

### Sentry

1. Sign up at https://sentry.io
2. Create project: **RakshEx Frontend** (Next.js)
3. Copy DSN to Vercel env var `NEXT_PUBLIC_SENTRY_DSN`
4. Create project: **RakshEx Backend** (Node.js)
5. Copy DSN to Render env var `SENTRY_DSN`

### UptimeRobot

1. Sign up at https://uptimerobot.com
2. Add monitors:
   - `https://rakshex.in` (every 5 min)
   - `https://api.rakshex.in/api/health` (every 5 min)
3. Set alert contacts (email, SMS, Slack webhook)

## Phase 6: Razorpay Payments (India)

Already configured in code. Just add production keys:

1. Go to https://dashboard.razorpay.com
2. **Settings** → **API Keys** → Generate new key
3. Add to Render env:
   ```
   RAZORPAY_KEY_ID=rzp_live_xxx
   RAZORPAY_KEY_SECRET=your_secret
   ```
4. Set webhook URL:
   ```
   https://api.rakshex.in/api/trpc/payment.handleWebhook
   ```
5. Add webhook secret to env var `RAZORPAY_WEBHOOK_SECRET`

## Phase 7: GitHub App (Optional, for CI/CD)

1. Go to https://github.com/settings/apps/new
2. Fill in:
   - **GitHub App name:** RakshEx Security Scanner
   - **Homepage URL:** https://rakshex.in
   - **Webhook URL:** https://api.rakshex.in/api/github/webhook
   - **Webhook secret:** Generate random string, add to env
   - **Permissions:**
     - Repository contents: Read-only
     - Pull requests: Read & write
     - Checks: Read & write
     - Commit statuses: Read & write
3. Generate private key (PEM file)
4. Copy contents to Render env var `GITHUB_APP_PRIVATE_KEY`
5. Copy App ID to `GITHUB_APP_ID`

## Phase 9: Backup Strategy

### Database Backups

**Render MySQL:**

- Daily automated backups (included in plan)
- Manual: `mysqldump` via Render dashboard shell

**PlanetScale:**

- Daily automated backups
- Point-in-time recovery (up to 7 days)

### Manual Backup Script

```bash
#!/bin/bash
# save as backup.sh
DATE=$(date +%Y%m%d_%H%M%S)
mysqldump -h $DB_HOST -u $DB_USER -p$DB_PASS rakshex > rakshex_backup_$DATE.sql
gzip rakshex_backup_$DATE.sql
aws s3 cp rakshex_backup_$DATE.sql.gz s3://rakshex-backups/
```

## Troubleshooting

### "Application error: client-side exception"

1. Check browser console for React errors
2. Verify `NEXT_PUBLIC_TS_API_URL` points to live backend
3. Check Sentry for stack traces

### "Cannot connect to database"

1. Verify `DATABASE_URL` format: `mysql://user:pass@host:3306/db`
2. Check Render MySQL is "Available" (not provisioning)
3. Test connection from Render shell: `mysql -h $DB_HOST -u $DB_USER -p`

### "Emails not sending"

1. Verify SMTP credentials in Render env
2. Check SendGrid/Mailgun dashboard for suppressed/bounced emails
3. Verify sender domain authentication

### "Build fails on Vercel"

1. Check build logs in Vercel dashboard
2. Ensure `pnpm-lock.yaml` is committed
3. Verify Node.js version compatibility (18+)
