## THE PROBLEM WITH "AI SECURITY"

Current "AI security" means:

- Prompt injection detection (one attack vector)
- Model alignment (research problem)
- Output filtering (reactive, not preventive)

**What's missing:** Runtime protection for AI agents in production.

When an AI agent is running:

- It makes API calls you don't see
- It loops infinitely until your budget is gone
- It leaks secrets you forgot existed
- It accesses endpoints you didn't authorize

No existing tool stops this.
