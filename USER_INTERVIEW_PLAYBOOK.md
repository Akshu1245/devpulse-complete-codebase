# RakshEx User Interview Playbook

> How to talk to beta users and extract actionable insights.
> Date: 2026-05-17

---

## INTERVIEW STRUCTURE (30 minutes)

### 1. Context (5 min)

```
"Tell me about what you're building."
"How do you use LLMs / AI APIs in your work?"
"What's your monthly API spend roughly?"
```

### 2. Current Pain (10 min)

```
"Walk me through the last time you had an issue with an AI agent."
"How do you currently monitor API costs?"
"Have you ever had an unexpected API bill? Tell me about it."
"How do you check for security issues in your API collections?"
```

### 3. RakshEx Experience (10 min)

```
"Walk me through your first time using RakshEx."
"What was the first finding you saw? What did you think?"
"Did you fix any finding because of RakshEx?"
"What's the most valuable thing RakshEx has done for you?"
"What's the most frustrating thing?"
"If you could change one thing, what would it be?"
```

### 4. Future (5 min)

```
"How likely are you to recommend RakshEx to a friend? (0-10)"
"If RakshEx disappeared tomorrow, what would you use instead?"
"What would make you pay $29/month for this?"
"Who else on your team should use this?"
```

---

## INTERVIEWER NOTES

### Do:

- Let the user talk 80% of the time
- Ask "tell me more about that"
- Take verbatim notes on their exact words
- Record with permission (for later analysis)

### Don't:

- Pitch features or defend the product
- Ask leading questions ("Don't you think X is useful?")
- Talk more than the user
- Rush to solutions

---

## SIGNAL EXTRACTION FRAMEWORK

After each interview, score the user on:

| Dimension                | Score (1-5)                        | Evidence |
| ------------------------ | ---------------------------------- | -------- |
| **Pain intensity**       | How much do they feel the problem? |          |
| **Product-market fit**   | How well does RakshEx solve it?    |          |
| **Onboarding ease**      | How smooth was first use?          |          |
| **Value clarity**        | Can they articulate the ROI?       |          |
| **Retention likelihood** | Will they still use in 30 days?    |          |
| **Referral likelihood**  | Would they recommend?              |          |

**Total score: /30**

- 25-30: Design partner — invest heavily
- 18-24: Potential user — address blockers
- < 18: Wrong segment — learn why, move on
