# RakshEx Public Beta Launch Sequence

> Step-by-step execution plan for going from private beta to public launch.
> Date: 2026-05-17

---

## PHASE 0: PRIVATE BETA (Weeks 1-4)

### Goal: Validate product with 25 friendly users

**Week 1**

- [ ] Invite 5 personal network developers
- [ ] Monitor onboarding funnel daily
- [ ] Fix critical blockers within 24 hours
- [ ] Conduct 3 user interviews

**Week 2**

- [ ] Invite 10 more from target ICP
- [ ] A/B test welcome view variations
- [ ] Measure activation rate (target: 60%)
- [ ] Fix top 3 user-reported pains

**Week 3**

- [ ] Invite 10 cold outreach developers
- [ ] Collect first NPS scores
- [ ] Document "saved me" stories
- [ ] Refine onboarding based on data

**Week 4**

- [ ] Analyze retention cohorts
- [ ] Calculate true positive rate
- [ ] Write beta retrospective
- [ ] Decide: ready for public beta?

**Exit Criteria:**

- Activation rate ≥ 50%
- 7-day retention ≥ 30%
- NPS ≥ 30
- Zero critical bugs
- 3+ "saved me" stories documented

## ROLLBACK TRIGGERS

**Pause launch if:**

- Critical bug affects > 10% of users
- Server error rate > 5%
- Extension crash rate > 2%
- Negative sentiment > 30% on social

**Full rollback if:**

- Security incident
- Data loss
- Legal issue
